## 二、技术选型

### 前端技术栈

| 层级 | 技术选型 | 选型理由 |
|------|----------|----------|
| 框架 | React 18 + TypeScript | 组件化开发，类型安全，生态成熟 |
| 构建工具 | Vite 5 | 极速冷启动，HMR 体验优秀 |
| 路由 | React Router v6 | 声明式路由，支持嵌套与懒加载 |
| 状态管理 | Zustand | 轻量简洁，API 优雅，适合中等规模项目 |
| 样式方案 | CSS Modules + CSS Variables | 作用域隔离，便于主题切换与古典美学变量管理 |
| 视觉效果 | SVG + Canvas 混合 | SVG 绘制印章、折扇等矢量图形，Canvas 绘制墨点热力图与长卷 |
| HTTP 请求 | Axios | 拦截器统一处理认证与错误 |
| 日期处理 | Day.js | 轻量级，支持中文本地化 |
| 字体方案 | 在线字体 + @font-face | 刻书体（"仿宋"/"KaiTi"备选）+ 行楷风格 |

### 后端技术栈

| 层级 | 技术选型 | 选型理由 |
|------|----------|----------|
| 运行时 | Node.js 20 LTS | 前后端语言统一，异步 I/O 适合高并发 |
| Web 框架 | Express.js | 生态最成熟，中间件丰富，灵活可扩展 |
| 数据库 | MongoDB 7 | 文档型数据库，灵活 Schema，适合作品/笔记等半结构化数据 |
| ODM | Mongoose 8 | 数据模型定义、校验、中间件钩子 |
| 认证 | JWT (jsonwebtoken + bcrypt) | 无状态认证，便于水平扩展 |
| 参数校验 | Joi / Zod | 运行时类型校验 |
| 日志 | Winston + Morgan | 结构化日志与 HTTP 请求日志 |
| 搜索 | MongoDB 全文索引 + 预留扩展位 | 初期用内置索引，后续可无缝接入 Elasticsearch |

## 四、核心设计原则

### 可扩展性设计

1. **模块化分层**：路由、控制器、服务、模型清晰分离，新增功能不侵入既有代码
2. **预留扩展位**：
   - 搜索层：当前 MongoDB 全文索引，可切换至 Elasticsearch/Meilisearch
   - 存储层：当前本地存储，可扩展 OSS/S3 存储封面与长卷图片
   - 通知层：预留消息通知接口，可接入邮件/推送
   - 社交层：预留作品关联与用户关系模型扩展位
3. **插件化**：统计计算、年度报告生成等功能以服务类封装，便于新增统计维度

### 性能设计

1. **前端**：
   - 路由级代码分割（React.lazy）
   - 长列表虚拟滚动
   - 墨点热力图与长卷使用 Canvas 离屏渲染
2. **后端**：
   - MongoDB 索引优化（用户ID、作品类型、创建时间等组合索引）
   - 统计数据预计算 + 缓存（Redis 预留）
   - 分页查询限制

### 安全设计

1. JWT 认证 + bcrypt 密码哈希
2. Helmet.js 安全头
3. 请求参数校验（Joi/Zod）
4. CORS 白名单
5. API 速率限制（express-rate-limit）

## 五、数据模型关系概览

```
User ──1:N──> Work (用户的作品收藏：剧集/书籍)
User ──1:N──> Note (用户的笔记批注)
User ──1:N──> Tag  (用户自定义标签)
Work ──N:M──> Tag  (作品打标签)
Note ──1:1──> Work (笔记关联作品)
Note ──1:1──> MoodColor (笔记关联一种中国色心情)
Work ──1:N──> Progress (观看/阅读进度记录)
```
